    Chapter 7: Software Engineering

    Terms Definitions
    Software Engineering Software Engineering is a discipline that involves the application of engineering principles and systematic methods to the design, development, testing, maintenance, and evolution of software systems.
    Integrated development environment (IDE) An Integrated Development Environment (IDE) is a software application that provides comprehensive facilities to programmers for software development.
    Association of Computing Machinery (ACM) The Association for Computing Machinery (ACM) is a professional organization dedicated to advancing the field of computing.
    Purpose of ACM The purpose of ACM is to advance computing as a science and a profession by fostering knowledge sharing, professional development, collaboration, and advocacy within the global computing community.
    Benefits of ACM membership ACM membership provides access to a global network of computing professionals, premier publications, conferences, and educational resources, fostering continuous learning and career advancement.
    Student ACM membership It is a membership category specifically designed for students interested in the field of computing, offering access to educational resources, networking opportunities, publications, and events tailored to support their academic and professional development in computing.
    Institute of Electrical and Ekectronics Engineers (IEEE) It is a professional organization dedicated to advancing technology for the benefit of humanity.
    Purpose of IEEE It is to advance technology for the benefit of humanity by fostering collaboration, disseminating technical knowledge, developing standards, and providing a global platform for professionals in electrical engineering, electronics, and computer science.
    Benefits of IEEE membership IEEE membership provides access to a global network of professionals, cutting-edge technical publications, industry-leading conferences, and valuable educational resources, fostering career growth and facilitating collaboration in the fields of electrical engineering, electronics, and computer science.
    Student IEEE membership Student IEEE membership offers students in electrical engineering, electronics, and computer science access to a valuable network, cutting-edge technical resources, and opportunities for professional development, enhancing their academic and career pursuits.
    Software Life Cycle: development phase: The software development life cycle phase involves systematically designing, coding, testing, and deploying software, encompassing the entire process from conception to the final release or maintenance
    Requirement Analysis It is the phase in the software development life cycle where stakeholders' needs and system requirements are systematically gathered, documented, and analyzed to form a comprehensive understanding of the software system's functionalities and constraints before the actual development process begins.
    Design Design in the context of software development refers to the process of creating a blueprint or plan that outlines the structure, architecture, and components of a software system, including how they interact and fulfill specified requirements, before the implementation phase begins.
    Implementation Implementation, in the context of software development, refers to the phase where the design specifications are translated into actual code or programming language, producing the executable software system according to the outlined architecture and requirements.
    Testing Testing, in the context of software development, is the systematic process of evaluating a software application to identify defects, ensure that it behaves as expected, and verify that it meets specified requirements.
    Modular Implementation Modular implementation in software development refers to the practice of breaking down a software system into smaller, independent, and cohesive modules or components that can be developed, tested, and maintained separately.
    Unified Modeling Language (UML) Unified Modeling Language (UML) is a standardized visual modeling language in the field of software engineering that provides a set of notations and diagrams to describe, design, and document the structure and behavior of a system.
    Use case A use case is a description of how a system interacts with an external entity (such as a user, another system, or an external process) to achieve a specific goal or functionality.
    Use case diagram A use case diagram is a type of diagram in the Unified Modeling Language (UML) that visually represents the interactions between different actors (external entities, such as users or systems) and a system in terms of use cases.
    Data relationships Data relationships refer to the associations and connections between different pieces of data in a system or database. These relationships define how data entities are related to each other, and they are a fundamental aspect of database design and management.
    one-to-one A one-to-one (1:1) relationship is a type of data relationship in which each record in one data table is directly associated with, at most, one record in another data table, and vice versa.
    one-to-many A one-to-many (1:N) relationship is a type of data relationship in which a single record in one data table can be associated with multiple records in another data table, but each record in the second table is associated with at most one record in the first table.
    many-to-many A many-to-many (N:N) relationship is a type of data relationship in which multiple records in one data table can be associated with multiple records in another data table.
    Design pattern A design pattern is a reusable and general solution to a recurring problem in software design. It represents a best practice or a template that can be applied to address common challenges in the development of software systems.
    Software quality assurance (SQA) Software Quality Assurance (SQA) refers to the systematic process of ensuring that software products and the processes used to develop them adhere to specified requirements, standards, and procedures.
    Software testing Software testing is the process of systematically evaluating a software application or system to identify and rectify any defects, errors, or discrepancies between expected and actual results.
    glass box testing Glass box testing, also known as White Box Testing or Clear Box Testing, is a software testing technique that examines the internal structure, logic, and code of a software application.
    black box testing Black box testing is a software testing technique where the internal structure, design, or implementation details of the software being tested are not known to the tester.
    beta testing Beta testing is the phase of software testing that occurs after alpha testing and before the official release of the software to the public.
    alpha testing Alpha testing is a phase of software testing that occurs before the software is released to a broader audience, typically involving internal testing by members of the development team or a dedicated testing team.
    User documentationUser documentation, often referred to as user manuals or user guides, is a set of documents created to provide information and instructions to end-users about how to effectively and efficiently use a software application, product, or system.
    System DocumentationSystem documentation refers to a comprehensive set of documents that provide detailed information about the design, architecture, functionality, and operation of a software system or an entire information system.
    Technical DocumentationTechnical documentation refers to a set of documents that provide detailed information about the technical aspects, specifications, and usage of a product, system, or process.
    Intellectual Property lawIntellectual Property (IP) law refers to a set of legal principles and regulations that govern the protection of intangible creations of the mind.
    Software LicenseA software license is a legal agreement or contract between the owner or licensor of a software program and the end-user or licensee. The license outlines the terms and conditions under which the software can be used, distributed, modified, or shared.
    Software DisclaimerA software disclaimer is a legal statement or notice provided by the software developer or distributor to clarify certain aspects of the software's use, limitations, and potential risks.
    Best defenses against software liability claimsOne of the best defenses against software liability claims is to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
